====== Не срабатывают GPO ======

Иногда бывает так-что при просмотре gpresult, политики применяются, но по факту не работают.
При этом в Журнале событий -> Система можно лицезреть подобные ошибки

<code>
Для домена SES нет доступного контроллера домена. Ошибка: 
Отсутствуют серверы, которые могли бы обработать запрос на вход в сеть...
</code>

Обычно подобные ошибки сопровождаются различными номерами 5719, 1054 и др.

Для решения данной проблемы можно применять следующие шаги:

  - Установить драйвера от производителя для сетевой карты, а не те которые по умолчанию устанавливает Windows
  - Изменить значение в реестре для отключения проверки состояния сетевого подключения для протокола TCP/IP (см. [[https://support.microsoft.com/ru-ru/help/239924/how-to-disable-the-media-sensing-feature-for-tcp-ip-in-windows | ссылку]])

<code>
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\Tcpip\Parameters
Добавьте следующую запись реестра в Parameters подраздел:
Имя: DisableDHCPMediaSense
Тип данных: REG_DWORD (логическое)
Значение: 1
</code>
